# RFC 8017 - PKCS #1: RSA密码学规范 Version 2.2

## 文档概述

RFC 8017是由IETF于2016年11月发布的信息性备忘录，标题为"PKCS #1: RSA Cryptography Specifications Version 2.2"。该文档取代RFC 3447，是当前最新的RSA标准版本。

**RFC 8017的主要特点**：

1. **变更控制转移**：将PKCS #1的变更控制权从RSA实验室转移到IETF
2. **更新安全建议**：废弃MD2、MD5等不安全的哈希算法
3. **推荐现代算法**：推荐使用SHA-256或更强的哈希函数
4. **保持技术稳定**：核心技术方案与RFC 3447保持一致
5. **当前标准**：是目前最新的RSA官方标准

RFC 8017代表了PKCS #1标准从RSA实验室向开放标准组织IETF的正式转移，标志着RSA标准进入了新的发展阶段。

## 历史背景

### 从v2.1到v2.2的演进

RFC 3447 (v2.1)在2003年发布后，在长达13年的时间里一直是RSA的主要标准。到2016年，基于以下原因发布了v2.2：

**主要更新动机**：

1. **哈希算法安全性**：MD5和SHA-1已被攻破，需要更新推荐算法
2. **标准控制权转移**：将控制权从RSA实验室转移到IETF
3. **安全建议更新**：反映最新的密码学研究成果
4. **算法标识符更新**：更新ASN.1对象标识符

### 完整版本演进

- **1998年3月**：RFC 2313 (v1.5) - 首次RFC标准化
- **1998年10月**：RFC 2437 (v2.0) - 引入OAEP
- **2003年2月**：RFC 3447 (v2.1) - 引入PSS
- **2016年11月**：RFC 8017 (v2.2) - 当前版本

## 核心方案

### 保持技术稳定

RFC 8017在技术方案上与RFC 3447保持一致：

**加密方案**：
- RSAES-OAEP（推荐）
- RSAES-PKCS1-v1_5（兼容）

**签名方案**：
- RSASSA-PSS（推荐）
- RSASSA-PKCS1-v1_5（兼容）

**辅助函数**：
- MGF1掩码生成函数
- 各种哈希函数支持

## 主要更新内容

### 哈希算法更新

**废弃的算法**：
- MD2：已被完全攻破
- MD5：存在严重碰撞攻击
- SHA-1：不再推荐用于新应用

**推荐的算法**：
- SHA-256（推荐）
- SHA-384
- SHA-512
- SHA-512/224
- SHA-512/256

### 安全建议更新

**密钥长度**：
- 最小推荐：2048位
- 中期保护：3072位
- 长期保护：4096位

**随机数生成**：
- 必须使用密码学安全的随机数生成器
- 符合NIST SP 800-90A等标准

### 算法标识符更新

**ASN.1对象标识符**：
- 更新了各种算法的OID
- 添加了新哈希算法的标识符
- 保持向后兼容

## 与RFC 3447的对比

### 技术方案

**保持不变**：
- RSAES-OAEP加密方案
- RSASSA-PSS签名方案
- RSAES-PKCS1-v1_5加密方案
- RSASSA-PKCS1-v1_5签名方案
- MGF1掩码生成函数
- 密钥语法和格式

**主要变化**：
- 废弃MD2、MD5哈希算法
- 更新安全建议和推荐参数
- 更新算法标识符
- 变更控制权转移到IETF

## 安全考虑

### 现代安全要求

**哈希函数选择**：
- 必须使用SHA-256或更强的哈希
- 避免使用SHA-1（已被攻破）
- 绝对禁止使用MD5和MD2

**密钥管理**：
- 使用足够长的密钥（至少2048位）
- 定期更新密钥
- 安全存储私钥

### 方案选择建议

**加密**：
- 新应用必须使用RSAES-OAEP
- 不要使用RSAES-PKCS1-v1_5

**签名**：
- 推荐使用RSASSA-PSS
- RSASSA-PKCS1-v1_5仍可接受但不推荐

## 总结

RFC 8017作为PKCS #1 Version 2.2，是当前最新的RSA密码学标准。该文档在保持RFC 3447技术方案稳定性的基础上，更新了安全建议以反映最新的密码学研究成果。

**主要特点**：

1. **当前标准**：是目前最新的RSA官方标准
2. **安全更新**：废弃不安全的哈希算法，推荐SHA-256或更强
3. **控制权转移**：将变更控制从RSA实验室转移到IETF
4. **技术稳定**：核心方案与v2.1保持一致
5. **向后兼容**：继续支持旧版方案以保持互操作性

**历史意义**：

RFC 8017标志着PKCS #1标准完全转移到IETF管理。从1991年RSA实验室首次发布PKCS #1，到2016年IETF接管标准控制权，这一转变反映了RSA算法从专利保护的商业技术向完全开放的互联网标准的演进历程。

**PKCS #1完整演进历程**：

- **1991年**：RSA实验室首次发布PKCS #1
- **1998年3月**：RFC 2313 (v1.5) - 首次RFC标准化
- **1998年10月**：RFC 2437 (v2.0) - 引入OAEP，应对安全漏洞
- **2003年2月**：RFC 3447 (v2.1) - 引入PSS，成为最广泛使用版本
- **2016年11月**：RFC 8017 (v2.2) - 控制权转移到IETF，当前版本

**现代应用建议**：

对于2026年的新应用开发：
- 必须使用RFC 8017作为参考标准
- 加密使用RSAES-OAEP配合SHA-256或更强的哈希
- 签名推荐使用RSASSA-PSS配合SHA-256或更强的哈希
- 密钥长度至少2048位，推荐3072位或4096位
- 考虑迁移到后量子密码算法（如NIST标准化的算法）
